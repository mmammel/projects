# Product Guidelines

## Visual Style
- **Minimalist & Modern:** The application should feature a clean and uncluttered interface. This includes generous whitespace, simple and legible typography, and subtle transitions between states to maintain a high-quality, modern feel.

## Interaction & Tone
- **Exploratory & Playful:** Interactions should feel tactile and responsive. Smooth animations should be used to provide visual feedback for user actions (like rotation or exploding faces), fostering a sense of fun and discovery.

## Code Standards
- **Concise & Functional:** Development should prioritize modularity and clear, descriptive naming for functions and variables.
- **Strategic Documentation:** Comments should be used sparingly, focusing on explaining the "why" behind complex logic or mathematical algorithms rather than describing what the code is doing.
